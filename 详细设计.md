## 2. 数据模型与枚举

### 2.1 枚举（models.py）

```python
class Asset(str, Enum):
    BTC = "BTC"; ETH = "ETH"; USDT = "USDT"

class OrderSide(str, Enum):
    BUY = "BUY"; SELL = "SELL"

class OrderStatus(str, Enum):
    OPEN = "OPEN"
    PARTIALLY_FILLED = "PARTIALLY_FILLED"
    FILLED = "FILLED"
    CANCELLED = "CANCELLED"

```

枚举均继承 `str`，便于直接 JSON 序列化和 SQL 字符串存储。

### 2.2 实体（dataclass）

| 实体 | 字段 | 关键不变量 |
| --- | --- | --- |
| `User` | username, password_hash, password_salt, created_at | 用户名 strip 后非空 |
| `Wallet` | owner, balances, frozen_balances | 余额非负，全部为 `Decimal` |
| `Order` | order_id, user_id, pair, side, price, quantity, remaining_quantity, status, created_at, sequence, order_type | 限价单 price > 0；市价单 price = 0；`0 ≤ remaining_quantity ≤ quantity` |
| `Trade` | trade_id, buy_order_id, sell_order_id, buyer_id, seller_id, pair, price, quantity, quote_amount, timestamp | quote_amount = price × quantity |
| `Block` | index, timestamp, previous_hash, transactions, block_hash | block_hash = SHA-256(payload) |
| `OrderPlacementResult` | order, trades, self_trade_blocked | 撮合结果聚合体 |
| `TradingPair` | base_asset, quote_asset | base ≠ quote |

### 2.3 金额标准化

```python
DECIMAL_SCALE = Decimal("0.00000001")   # 8 位小数
ZERO = Decimal("0")

def normalize_decimal(value):
    decimal_value = Decimal(str(value))
    if not decimal_value.is_finite():
        raise ValidationError(...)
    return decimal_value.quantize(DECIMAL_SCALE, rounding=ROUND_DOWN)

def ensure_positive(value, field_name):
    if value <= ZERO: raise ValidationError(...)
```

**关键设计**：

- 量化使用 `ROUND_DOWN`，**不会向上凑整**金额，避免冻结/结算虚增。
- 拒绝 `NaN`、`Infinity`、`-Infinity`、字符串中混入字母等非法输入。
- 所有外部输入（API 参数、SQL 读出的字符串）进入业务层之前都需要经过 `normalize_decimal`。

### 2.4 订单状态机

```text
              register, validate, freeze
                       │
                       ▼
                    ┌──────┐
                    │ OPEN │
                    └──┬───┘
                       │ 部分成交
                       ▼
              ┌─────────────────┐
              │PARTIALLY_FILLED │
              └────────┬────────┘
                       │ 全部成交               │ 撤销
                       ▼                        ▼
                   ┌────────┐               ┌──────────┐
                   │ FILLED │               │ CANCELLED│
                   └────────┘               └──────────┘
```

实现：`Order.refresh_status()`：

```python
if remaining_quantity == ZERO:        status = FILLED
elif remaining_quantity < quantity:   status = PARTIALLY_FILLED
else:                                 status = OPEN
```

撤单单独由 `MatchingEngine.cancel_order()` 触发置 `CANCELLED`。

### 3.2 PRAGMA 设置

```python
conn = sqlite3.connect(db_path, check_same_thread=False)
conn.row_factory = sqlite3.Row
conn.execute("PRAGMA journal_mode=WAL")
conn.execute("PRAGMA foreign_keys=ON")
```

- `check_same_thread=False`：允许 `ThreadingHTTPServer` 跨线程共用连接。
- WAL：减小读写阻塞（单进程内）。
- 外键开启：保留依赖语义（当前 schema 未声明显式 FK，预留扩展能力）。

### 3.3 schema 迁移策略

`_migrate_schema(conn)` 当前实现：

```python
columns = {row["name"] for row in conn.execute("PRAGMA table_info(orders)")}
if "order_type" not in columns:
    conn.execute("ALTER TABLE orders ADD COLUMN order_type TEXT NOT NULL DEFAULT 'LIMIT'")
    conn.commit()
```

后续新增列时，依照同一模式追加：

1. 用 `PRAGMA table_info(<table>)` 读取实际列集合。
2. 针对缺失列执行幂等 `ALTER TABLE`。
3. 严禁 `DROP COLUMN`，避免老库数据丢失。

### 3.4 重置策略

`clear_db(conn)`：

```sql
DELETE FROM pending_transactions;
DELETE FROM blocks;
DELETE FROM trades;
DELETE FROM orders;
DELETE FROM wallets;
DELETE FROM sessions;
DELETE FROM users;
```

按"子表 → 主表"顺序删除（即使当前未声明外键，也保留语义）。

```python
class PasswordHasher:
```

#### 4.2.2 操作伪代码

```python
def deposit(user, asset, amount):
    validate_asset(asset)
    amount = normalize_decimal(amount); ensure_positive(amount)
    wallet.balances[asset] += amount
    flush_asset(user, asset)

def withdraw(...):
    if balances[asset] < amount: raise InsufficientBalanceError
    balances[asset] -= amount; flush_asset(...)

def freeze(...):
    if balances[asset] < amount: raise InsufficientBalanceError
    balances[asset] -= amount
    frozen_balances[asset] += amount; flush_asset(...)

def unfreeze(...):
    if frozen_balances[asset] < amount: raise InsufficientFrozenBalanceError
    frozen_balances[asset] -= amount
    balances[asset] += amount; flush_asset(...)

def consume_frozen(...):
    if frozen_balances[asset] < amount: raise InsufficientFrozenBalanceError
    frozen_balances[asset] -= amount; flush_asset(...)   # 注意：不增加 balances
```

#### 4.2.3 持久化策略

- `_flush_asset(user, asset)`：单行 `INSERT OR REPLACE INTO wallets`，避免 race。
- `create_wallet_for_user`：在 `conn` 模式下为该用户的所有支持币种插入 `0` 行（`INSERT OR IGNORE`）。

### 4.3 撮合模块（engine.py）

#### 4.3.1 OrderBook

```python
class OrderBook:
    def add(self, order) -> None
    def get_orders(self, pair, side) -> List[Order]
    def remove(self, order) -> bool
    def snapshot(self, pair) -> dict
```

排序规则（每次插入时立即排序）：

```python
if side == BUY:
    book.sort(key=lambda o: (-o.price, o.sequence))   # 价格降序，时间升序
else:
    book.sort(key=lambda o: (o.price, o.sequence))    # 价格升序，时间升序
```

数据结构：`DefaultDict[pair_symbol, List[Order]]`，对每个 (pair, side) 维护独立列表；列表头部即"对手簿最优价"。

#### 4.3.2 限价单流程

```python
def place_limit_order(user, pair, side, price, quantity):
    validate_user(user); validate_pair(pair)
    price = normalize_decimal(price); ensure_positive(price)
    quantity = normalize_decimal(quantity); ensure_positive(quantity)

    order = Order(
        order_id = f"O{next(order_id_seq):06d}",
        side, price, quantity,
        remaining_quantity = quantity,
        status = OPEN,
        sequence = next(order_seq),
        order_type = LIMIT,
    )

    freeze_order_funds(order)         # BUY: USDT = price*qty；SELL: BTC = qty
    trades, self_blocked = match_order(order)
    if order.is_active(): order_book.add(order)
    persist_order(order); persist trades and makers
    return OrderPlacementResult(order, trades, self_blocked)
```

#### 4.3.3 市价单流程（关键）

```python
def place_market_order(user, pair, side, quantity):
    validate_user(user); validate_pair(pair)
    quantity = normalize_decimal(quantity); ensure_positive(quantity)

    opposite_book = order_book.get_orders(pair, opposite(side))
    matchable = [m for m in opposite_book if m.user_id != user]
    available = sum(m.remaining_quantity for m in matchable)
    if available < quantity:
        raise EmptyOrderBookError(...)            # 不冻结资金，直接拒绝

    if side == BUY:
        # 预走对手簿，精确算出所需 quote 总额
        remaining = quantity; required_quote = ZERO
        for m in matchable:
            take = min(remaining, m.remaining_quantity)
            required_quote += m.price * take
            remaining -= take
            if remaining <= ZERO: break
        wallet.freeze(user, pair.quote_asset, required_quote)
    else:
        wallet.freeze(user, pair.base_asset, quantity)

    order = Order(... order_type=MARKET, price=ZERO ...)
    trades, self_blocked = match_order(order)
    if order.is_active(): order.status = CANCELLED   # 兜底：理论不可达
    persist...
```

**为什么要预走？** 因为撮合时市价买单会按 maker 的不同价格逐档成交，如果只按"假定最坏价格 × 数量"冻结，会要么超额冻结浪费可用余额，要么金额不足导致 `consume_frozen` 在中途失败。预走可以一次性锁定精确总额。

#### 4.3.4 撮合主循环

```python
def match_order(taker):
    trades = []; self_trade_blocked = False
    book = order_book.get_orders(taker.pair, opposite(taker.side))
    pointer = 0
    while taker.remaining > 0 and pointer < len(book):
        maker = book[pointer]
        if not is_price_crossed(taker, maker): break
        if maker.user_id == taker.user_id:
            if taker.order_type == MARKET:
                pointer += 1; continue            # 跳过自己
            self_trade_blocked = True; break       # 限价单停止
        qty = min(taker.remaining, maker.remaining)
        execution_price = maker.price             # 成交价取 maker
        trade = execute_trade(taker, maker, qty, execution_price)
        trades.append(trade)
        blockchain.add_trade(trade)               # 上链 + 自动封块
        if maker.remaining == 0: book.pop(pointer)
        # 成交满量的 maker 才 pop；未满量保持原位置以保持下次扫描效率
    taker.refresh_status()
    return trades, self_trade_blocked

def is_price_crossed(taker, maker):
    if taker.order_type == MARKET: return True
    if taker.side == BUY:  return taker.price >= maker.price
    else:                  return taker.price <= maker.price
```

#### 4.3.5 成交结算

```python
def execute_trade(taker, maker, qty, exec_price):
    buy_order, sell_order = (taker, maker) if taker.side==BUY else (maker, taker)
    quote_amount = exec_price * qty

    wallet.consume_frozen(sell_order.user_id, base_asset, qty)
    wallet.deposit       (buy_order.user_id,  base_asset, qty)
    wallet.consume_frozen(buy_order.user_id,  quote_asset, quote_amount)
    wallet.deposit       (sell_order.user_id, quote_asset, quote_amount)

    if buy_order.price > exec_price:
        rebate = (buy_order.price - exec_price) * qty
        if rebate > 0:
            wallet.unfreeze(buy_order.user_id, quote_asset, rebate)

    taker.remaining -= qty;  maker.remaining -= qty
    taker.refresh_status();  maker.refresh_status()
    return Trade(trade_id=f"T{next(trade_id_seq):06d}", ...)
```

**返还差额**仅适用限价买单（市价单 `price=0`，条件不满足，不返还）。

#### 4.3.6 撤单

```python
def cancel_order(user, order_id):
    order = orders.get(order_id) or raise OrderNotFoundError
    if order.user_id != user: raise ValidationError
    if not order.is_active(): raise OrderNotCancellableError

    if order.side == BUY:
        refund_asset  = pair.quote_asset
        refund_amount = order.price * order.remaining_quantity
    else:
        refund_asset  = pair.base_asset
        refund_amount = order.remaining_quantity

    if refund_amount > 0:
        wallet.unfreeze(order.user_id, refund_asset, refund_amount)
    order_book.remove(order)
    order.status = CANCELLED
    persist_order(order)
    return order
```

#### 4.4.3 Block.compute_hash

```python
def compute_hash(self):
    payload = {
        "index": self.index,
        "timestamp": self.timestamp.isoformat(),
        "previous_hash": self.previous_hash,
        "transactions": self.transactions,
    }
    serialized = json.dumps(payload, sort_keys=True, ensure_ascii=False)
    return hashlib.sha256(serialized.encode()).hexdigest()
```

`sort_keys=True` 保证序列化稳定，跨 Python 版本可重现。

#### 4.4.4 自动封块

```python
def add_transaction(tx):
    pending_transactions.append(deepcopy(tx))
    persist pending tx (INSERT INTO pending_transactions)
    if len(pending_transactions) >= block_capacity:
        seal_pending_transactions()

def seal_pending_transactions():
    if not pending: raise EmptyBlockError
    prev = chain[-1]
    block = Block(index=len(chain), timestamp=clock(),
                  previous_hash=prev.block_hash,
                  transactions=deepcopy(pending))
    block.block_hash = block.compute_hash()
    chain.append(block); pending.clear()
    persist_block(block); DELETE FROM pending_transactions
    return deepcopy(block)
```

#### 4.4.5 链校验

```python
def validate_chain():
    if not chain: raise ChainValidationError("链为空")
    for i, block in enumerate(chain):
        if not block.transactions: raise ChainValidationError(f"区块 {i} 交易为空")
        if block.block_hash != block.compute_hash():
            raise ChainValidationError(f"区块 {i} 哈希不匹配")
        if i == 0:
            if block.previous_hash != "0"*64:
                raise ChainValidationError("创世区块 previous_hash 非法")
        else:
            if block.previous_hash != chain[i-1].block_hash:
                raise ChainValidationError(f"区块 {i} previous_hash 与前一区块不一致")
    return True
```


异常翻译：


### 5.3 鉴权流程

```python
def _authenticate_or_fail(self) -> tuple[user, token] | None:
    token = _extract_bearer_token()
    if token is None:
        _send_json({...}, 401); return None
    try:
        user = STATE.auth_service.get_user_by_token(token)
    except DEXError:
        _send_json({...}, 401); return None
    return user.username, token

def _extract_bearer_token(self) -> str | None:
    header = self.headers.get("Authorization", "")
    if not header.startswith("Bearer "): return None
    return header[len("Bearer "):].strip() or None
```

**关键不变量**：钱包/下单/撤单/登出接口的"操作主体"始终由 token 解析得到，请求体里的 `username` 一律忽略。

测试用例 `test_wallet_uses_token_user_not_body_username` 强制守护此不变量。

```python
def _send_static(self, rel_path):

    abs_path = os.path.realpath(os.path.join(STATIC_DIR, safe))
```

### 5.5 自动重启

```python
def _watch_sources(server, reload_event):
    initial = _source_signature()  # {path: mtime}
    while not reload_event.is_set():
        time.sleep(0.8)
        current = _source_signature()
        changed = [p for p in initial|current if initial.get(p) != current.get(p)]
        if changed:
            reload_event.set()
            server.shutdown()
            return

# main():
finally: server.server_close()
if reload_event.is_set():
```

## 7. 异常与错误处理

### 7.1 异常 → HTTP 状态码映射

| 异常 | HTTP | 触发场景 |
| --- | --- | --- |
| `ValidationError` | 400 | 参数非法、币种不支持、金额非法 |
| `UserAlreadyExistsError` | 400 | 注册重复 |
| `UserNotFoundError` | 400 | 登录用户不存在 |
| `InvalidCredentialsError` | 400 / 401 | 密码错误 / token 失效（401 由 `_authenticate_or_fail` 显式返） |
| `InsufficientBalanceError` | 400 | 余额不足、提现失败 |
| `InsufficientFrozenBalanceError` | 400 | 冻结不足、解冻/扣减失败 |
| `UnsupportedAssetError` | 400 | 操作未支持币种 |
| `UnsupportedTradingPairError` | 400 | 下单交易对不支持 |
| `OrderNotFoundError` | 400 | 撤单订单不存在 |
| `OrderNotCancellableError` | 400 | 已成交/已撤销订单撤单 |
| `EmptyOrderBookError` | 400 | 市价单对手方流动性不足 |
| `EmptyBlockError` | 400 | 手动封块时无待打包交易 |
| `ChainValidationError` | 400 | 校验链时哈希错误 |
| 缺/错 token | 401 | `_authenticate_or_fail` |
| 未知路径 | 404 | `do_GET` / `do_POST` 兜底 |
| `KeyError` / `ValueError` | 400 | JSON 字段缺失或非法 |

## 9. 性能与限制

### 9.1 复杂度

| 操作 | 时间复杂度 |
| --- | --- |
| 注册 / 登录 | O(1) + 一次 PBKDF2 |
| 充值 / 提现 / 冻结 / 解冻 | O(1) + 一次 SQL |
| 限价单提交 | O(K + log N)，K 为成交数，N 为对手簿大小（排序使用 list.sort 是 O(N log N)） |
| 市价单提交 | O(N) 预走 + O(K) 成交 |
| 撤单 | O(N)（线性扫描列表） |
| 链校验 | O(B + T) |

### 9.2 已知限制

- 撮合中途异常无事务回滚。
- SQLite 不适合多机分布式部署。
- token 永不过期，会无限累积。
- watcher 线程基于 mtime 轮询，对超大代码库可能引入感知延迟。

### 10.2 ID 编码规则

- 订单 ID：`O` + 6 位零填充序号，例如 `O000001`。
- 成交 ID：`T` + 6 位零填充序号，例如 `T000001`。
- 区块 index：从 0 起的整数。

### 10.3 字段命名规范

- 数据库列名采用 `snake_case`（`order_id`、`block_index`）。
- Python 属性与字段保持一致命名，便于 `INSERT OR REPLACE` 时直接展开。
- API JSON 字段对外暴露 `lowerCamelCase`？— **本项目仍使用 `snake_case`，与内部模型对齐**，便于教学时阅读。
